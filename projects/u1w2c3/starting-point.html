Fluid Layouts

Fluid layouts are basically websites that scale along with the screen size. Typically, most websites use a fixed layout 1, which means that the width of the page is a set number of pixels, typically 960 these days. Fluid layouts on the other hand use units that scale with the size of the page, either percentages of the screen (the usual fluid, also known as liquid design) or in ems, which is a unit that is relative to the user's font size (this is referred to as elastic design)2. In both of these cases, the webpage will look the same no matter how you resize you browser or what device you use to visit the site.

Responsive Design

Fluid layouts are part of a more general design process called responsive design. The layout will grow or shrink in response to the user's screen size. But there are other tricks one can use to alter the layout of a webpage based on the user's environment: media queries3. Media queries are bits of CSS code that only activate when certain conditions are met on the user's side, for instance the window size is within a certain range. This allows feats such as moving a sidebar to the top of the page when the screen is too narrow (like on a mobile device) or rearrange a grid of pictures to adapt their size to the available screen real estate and neatly occupy the optimal surface4.

Reflections

Responsive design is definitely a lot more work than a traditional, fixed width fire-and-forget approach, but it has so many advantages in terms of usability on different medium that it's hard not to justify the effort in my opinion. With the spread of smartphones and tablets, I think it's essential that your website stays usable on all devices if you want it to be successful. I had a lot of fun using media queries; they have a magical feel to them. As far as fluid/elastic layouts are concerned, I'm all for them. The idea of using fixed quantities for things like width and height and so on really doesn't sit well with me in the first place. Using non-fixed width does introduce a bit of trickiness, but I think it's absolutely worth it, especially given the increase of types of Web-capable devices we've had over the last few years.
